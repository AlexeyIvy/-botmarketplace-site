# Глоссарий и определения

Этот документ фиксирует термины, чтобы в ТЗ, коде и обсуждениях они понимались одинаково.

## Термины

- **MVP** — минимально жизнеспособная версия продукта (первый рабочий контур, проверка гипотезы).
- **Bybit Demo Trading** — режим демо-торговли Bybit, отдельная среда с отдельными endpoint’ами/ключами (используем в MVP).
- **Environment (env)** — окружение биржи: `demo` или `real` (в MVP real выключен флагом).
- **Category** — тип рынка/продукта в Bybit API V5 (в MVP: `linear`).
- **Linear** — деривативы USDT-margined (USDT perpetual и др.); это дефолт для MVP.
- **Symbol** — торговый инструмент (например, `BTCUSDT`).
- **Bot** — сущность “торговый бот” в нашей системе: конфигурация рынка + стратегия + риск-настройки.
- **Bot Run** — конкретный запуск бота во времени (start → работа → stop/timeout/error).
- **Strategy / Стратегия** — набор правил входа/выхода и риск-параметров.
- **Strategy Spec (DSL)** — структурированное представление стратегии (JSON-формат), которое генерирует AI и/или редактирует пользователь.
- **Intent** — “намерение” стратегии (например “открыть long”), которое проходит через слой Risk/Execution.
- **Execution layer** — слой, преобразующий intent в конкретные биржевые операции (place order/cancel/update SLTP).
- **Risk guard** — правила безопасности: лимиты объёма, лимиты убытка, лимиты частоты, стоп-кнопка.
- **SL/TP** — Stop Loss / Take Profit; в MVP обязательны.
- **Trading Stop (Bybit)** — механизм биржи для установки TP/SL/Trailing на позицию (в MVP это режим по умолчанию).
- **Idempotency** — свойство операций “повторный запрос не создаёт дубликат” (важно для place/cancel).
- **Reconciliation** — восстановление и сверка состояния (ордера/позиции) после реконнекта/сбоев.

## Соглашения

- В документации “обязательное” = MUST (без этого задача не считается выполненной).
- Все публичные идентификаторы ресурсов в API/URL планируем как UUID/ULID (не последовательные числа).
- Документация проекта — только на русском (RU), интерфейс продукта — RU+EN.
