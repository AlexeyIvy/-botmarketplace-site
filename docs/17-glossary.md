# Glossary (MVP)

## Термины

- Strategy (стратегия): декларативная конфигурация, описывающая вход/выход/риск/guards; валидируется схемой.
- Bot (бот): рантайм-исполнитель одной стратегии для одного пользователя и (в MVP) одного symbol; реализует state machine.
- Signal (сигнал): событие, инициирующее вход/выход или изменение состояния (webhook/ручной/внутренний).
- Guard (предохранитель): лимиты и правила безопасности (maxOpenPositions, maxOrdersPerMinute, pauseOnError).
- Reconciliation: сверка локального состояния бота с фактическими позициями/ордерами на бирже после рестарта/реконнекта.
- Demo-first: торговые операции выполняются только в demo окружении до отдельного решения о проде.

## Идентификаторы

- orderLinkId: пользовательский идентификатор ордера на Bybit; должен быть уникальным и не длиннее 36 символов. [web:358]
- intentId: внутренний идентификатор “намерения” (создать/изменить/закрыть), к которому привязываются попытки и итоговые orderId.
- ULID: тип идентификатора, лексикографически сортируемый; канонически кодируется строкой из 26 символов. [web:840]

## Указатели ошибок

- JSON Pointer: строковый синтаксис, указывающий на конкретное поле в JSON документе (пример: `/risk/riskPerTradePct`). [web:839]
